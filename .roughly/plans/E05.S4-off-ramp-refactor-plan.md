### T5: Create `docs/adrs/ADR-012-runtime-shared-procedural-references.md` (~5 min)
**Files:** `docs/adrs/ADR-012-runtime-shared-procedural-references.md`
**Action:** Create the ADR documenting the runtime-shared-reference pattern.
**Details:**
- Format: follow ADR-009 depth (sync-maintenance decision, not simple precedent).
- Header:
  ```
  # ADR-012: Runtime-shared procedural references

  **Date:** 2026-05
  **Status:** Accepted
  **Decider:** Nick Kirkes

  ---
  ```
- Sections:
  - **Context** — cite the E04.S3 line-cap binding at `skills/fix/SKILL.md` 300/300, E05.S2 cap pressure on `agents/doc-writer.md` (closed by E05.S1's 500→650 cap raise), the off-ramp candidate recorded at E04 epic L566, and the byte-identical-or-near-identical duplication of ABORT HANDLING and Stage 8 prose across `skills/build/SKILL.md` and `skills/fix/SKILL.md`. State the forcing function: any v0.1.6+ fix-touching work was structurally blocked.
  - **Decision** — Procedural prose duplicated across `skills/build/SKILL.md` and `skills/fix/SKILL.md` is extracted to authoritative files under `skills/shared/<name>.md` and referenced via runtime `Read \`skills/shared/<file>.md\`` directive at the section head. The orchestrator reads the shared file when reaching the section and applies the procedure documented there. Pipeline-specific divergence is handled via inline conditional prose ("When invoked from /roughly:build: X. When invoked from /roughly:fix: Y."), not template substitution. Distinct from ADR-003's sync-reference pattern, which covers *static context* (e.g., `agents/agent-preamble.md`) inlined verbatim in agents and manually kept in sync — ADR-012 is *runtime-loaded* procedural reference, not copy-and-sync.
  - **Consequences**
    - **Positive:** Recovers ~33 lines per consumer (build/fix), closes the 300/300 binding state and provides ~30-line headroom for downstream Stage-6/Stage-8 edits (E05.S6 AC3/AC4). Single source of truth for procedural prose — new content lands in one place. Pipeline-conditional divergence is readable inline (no template engine, no out-of-band substitution).
    - **Negative:** Adds a "must update both consumers + shared file" surface — a future contributor may edit Stage 8 prose inline in `skills/build/SKILL.md` without realizing the shared reference is authoritative, producing silent drift. Mitigated by AC5 drift check in `.claude/hooks/verify-all.sh` (path-presence + content-duplication phrase scan). Runtime `Read` adds one extra tool call per section entry.
    - **Neutral:** Shared files are documentation-class (no `disable-model-invocation` frontmatter required since they are not skills); implementer chooses whether to add YAML frontmatter — current E05.S4 implementation uses no frontmatter, since the files contain pure procedural prose and are loaded by the orchestrator, not invoked by the model.
  - **Alternatives Considered**
    - **Sync-reference per ADR-003 pattern.** Rejected — inlines the prose verbatim in both consumers; does not recover lines and re-creates the cap pressure that triggered the off-ramp.
    - **Per-stage extraction (extract every stage, not just Stage 8 + ABORT HANDLING).** Rejected — over-fragments the consumer; preamble + Stage 1 extraction explicitly deferred per OQ11 until the next forcing function.
    - **Bigger cap-bump (raise the 300-line SKILL.md cap).** Rejected — the duplication itself was technical debt; raising the cap masks the underlying problem and leaves drift risk in place.
  - **Forward References**
    - ADR-003 — related shared-reference pattern (copy-and-sync for static context); read alongside this ADR to understand which pattern applies for which case.
    - E05.S5 — first downstream story landing new content (2-commit-window ABORT HANDLING entry) in the shared file post-extraction.
    - E05.S6 AC3 (cubic-iteration termination criteria in Stage 6) and AC4 (plan-implementation drift framing in Stage 8) — landed in the shared file after E05.S4 lands.
**Verify:** `test -f docs/adrs/ADR-012-runtime-shared-procedural-references.md && grep -q '^# ADR-012:' docs/adrs/ADR-012-runtime-shared-procedural-references.md && grep -q '^## Context' docs/adrs/ADR-012-runtime-shared-procedural-references.md && grep -q '^## Decision' docs/adrs/ADR-012-runtime-shared-procedural-references.md`
